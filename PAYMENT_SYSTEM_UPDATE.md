## 🔄 منطق العمل

### عند تأكيد الدفع:
1. ✅ تحديث حالة الدفع إلى `completed`
2. ✅ تفعيل وصول الطالب للكورس (تحديث enrollment)
3. ✅ إرسال إشعار للطالب بالموافقة
4. ✅ تسجيل معرف المقدم الذي راجع الدفع
5. ✅ تسجيل تاريخ المراجعة

### عند رفض الدفع:
1. ✅ تحديث حالة الدفع إلى `failed`
2. ✅ حفظ سبب الرفض
3. ✅ إرسال إشعار للطالب مع السبب
4. ✅ عدم تفعيل وصول الطالب للكورس

## 📱 تجربة المستخدم

### مقدم الخدمة:

1. **إعداد بيانات الدفع:**
   - يذهب إلى الإعدادات
   - يدخل بيانات المحفظة/البنك
   - يحفظ الإعدادات

2. **مراجعة المدفوعات:**
   - يفتح شاشة المدفوعات
   - يرى قائمة بجميع المدفوعات
   - يفلتر المدفوعات المعلقة
   - يفتح صورة الإيصال للتحقق
   - يضغط "تأكيد" أو "رفض"

3. **عند التأكيد:**
   - يتم تفعيل الطالب فوراً
   - يظهر إشعار نجاح

4. **عند الرفض:**
   - يكتب سبب الرفض
   - يتم إرسال الإشعار للطالب

### الطالب (من جانب الطالب):

1. **عند الدفع:**
   - يرى بيانات الدفع (محفظة/بنك)
   - يقوم بالتحويل
   - يرفع صورة الإيصال
   - يدخل رقم العملية
   - ينتظر الموافقة

2. **عند الموافقة:**
   - يستلم إشعار بالموافقة
   - يتم تفعيل وصوله للكورس

3. **عند الرفض:**
   - يستلم إشعار مع السبب
   - يمكنه إعادة المحاولة

## 🚀 خطوات التشغيل

### 1. تحديث قاعدة البيانات:
```sql

psql -U postgres -d wasla_db -f database/update_payment_system.sql
```

أو من Supabase Dashboard:
- افتح SQL Editor
- انسخ محتوى `database/update_payment_system.sql`
- شغل الكود

### 2. إعادة تشغيل التطبيق:
```bash
flutter clean
flutter pub get
flutter run
```
