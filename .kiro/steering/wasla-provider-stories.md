# 🧑‍🏫 User Stories - مقدم الخدمة (Course Provider)

## 🎯 الهدف
إنشاء كورسات، إدارتها، متابعة الطلاب، وإصدار الشهادات.

---

## 🟢 الحساب والمصادقة

### US-P-001: إنشاء حساب مقدم خدمة
**كمقدم خدمة** أريد إنشاء حساب لبدء نشر الكورسات

**المتطلبات:**
- إدخال البيانات الأساسية
- التحقق من البريد الإلكتروني
- إضافة معلومات البنك (اختياري)
- الموافقة على الشروط

**الحقول المطلوبة:**
```
- email (فريد)
- password (آمن)
- name
- phone
- specialization
- bio
- bank_account (اختياري)
```

---

### US-P-002: تسجيل الدخول
**كمقدم خدمة** أريد تسجيل الدخول للوصول للوحة التحكم

**المتطلبات:**

- التحقق من البيانات
- حفظ جلسة العمل
- إعادة توجيه للوحة التحكم

## 💳 نظام الدفع المحلي - آلية العمل

### تدفق عملية الدفع:

1. **الطالب:**
   - يقوم بالتحويل البنكي أو الدفع عبر المحفظة
   - يرفع صورة الإيصال في التطبيق
   - يدخل رقم العملية والمعلومات المطلوبة

2. **مقدم الخدمة:**
   - يستقبل إشعار بطلب دفع جديد
   - يراجع صورة الإيصال والمعلومات
   - يتحقق من صحة الدفع
   - يقبل أو يرفض الطلب مع ذكر السبب

3. **النظام:**
   - عند القبول: يفعل وصول الطالب للكورس تلقائياً
   - عند الرفض: يرسل إشعار للطالب مع السبب
   - يحدث الإحصائيات والأرباح

### معلومات الدفع المطلوبة من المقدم:

```json
{
  "wallet_number": "05xxxxxxxx",
  "wallet_owner_name": "أحمد محمد",
  "bank_name": "البنك الأهلي",
  "bank_account_number": "SA12345678901234567890",
  "bank_account_owner_name": "أحمد محمد علي",
  "iban": "SA12345678901234567890",
  "additional_info": "يرجى كتابة اسم الكورس في وصف التحويل"
}
```

### حالات الدفع:

| الحالة | الوصف | الإجراء |
|-------|-------|---------|
| **pending** | قيد المراجعة | ينتظر مراجعة المقدم |
| **completed** | مكتمل | تم القبول وتفعيل الوصول |
| **failed** | مرفوض | تم الرفض مع ذكر السبب |
| **refunded** | مسترد | تم استرداد المبلغ |
